# 代码规范

# 组件中的data

组件的 data 必须是一个函数。当在组件中使用 data 属性的时候 (除了 new Vue 外的任何地方)，它的值必须是返回一个对象的函数。

    export default {
        data () {
            return {
                foo: 'bar'
            }
        }
    }

# 为v-for设置键值

总是用 key 配合 v-for。

    <ul>
        <li v-for="todo in todos" :key="todo.id">{{ todo.text }}</li>
    </ul>

# 指令缩写

都用指令缩写 (用 : 表示 v-bind: 和用 @ 表示 v-on:)

# 单文件组件的顶级元素的顺序

单文件组件应该总是让<template> 、<script>、和 <style>标签的顺序保持一致。

    <!-- ComponentA.vue -->
    <template>...</template>
    <script>...</script>
    <style>...</style>

# 总是使用 === 精确的比较操作符，避免在判断的过程中，由 JavaScript 的强制类型转换所造成的困扰

# 统一使用单引号(‘)，不使用双引号(“)。这在创建 HTML 字符串非常有好处

    var msg = 'This is some HTML <div class="makes-sense"></div>';
1

# HTML 引号

使用双引号(“”) 而不是单引号(”) 。

不推荐：

    <div class='news-article'></div>
1

推荐：

    <div class="news-article"></div>
1

# 避免直接修改component中的组件，如需修改，最好是遵循往组件内加功能，不修改原有功能的原则（避免其他已经使用该组件的页面出现问题）

# 代码缩进

统一使用四个空格进行代码缩进（一个tab = 四个空格）

# 标签嵌套

段落元素与标题元素只能嵌套内联元素

不推荐：

    <h1><div></div></h1>
    <p><div></div><div></div></p>
1
2

推荐：

    <h1><span></span></h1>
    <p><span></span><span></span></p>
